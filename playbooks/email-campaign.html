Overview

This playbook documents the end-to-end workflow for producing professional email outreach campaigns at The Cozm. It covers every step from finding target companies to sending personalised emails at scale.

The Christmas 2025 campaign serves as the reference example, with 269 recipients across major organisations including adidas, Fragomen, Siemens, KION, Dematic, and Santa Fe.

8-Step Workflow

1. Research (Gemini) → 2. Build List (Google Sheets) → 3. Validate (Emailable) → 4. Find Image (Pixabay)
↓
5. Host Image (Cloudinary) → 6. Design Email (HTML) → 7. Test Send → 8. Full Send (SendGrid)

Step 1: Research Target Companies

Using Gemini Deep Research

The Gemini Deep Research tool conducts comprehensive research to identify target companies and contacts in specific markets.

Example Queries:

"List the top 50 companies in Germany with global mobility programmes.
Include company name, headquarters location, industry, and typical
job titles for mobility managers."
"Identify major multinational corporations with offices in [Country]
that would require A1/Certificate of Coverage compliance.
Include HR and Global Mobility contact patterns."

Best Practices

  • Be specific about the industry or sector
  • Request contact title patterns (e.g., "Global Mobility Manager", "HR Director")
  • Ask for company size and employee count where relevant
  • Save research results to reference later

Step 3: Validate Email Addresses (Emailable)

⚠️ CRITICAL: NEVER send to unverified email addresses. Invalid emails damage sender reputation and can result in being blacklisted.

Verification Commands

Single Email:

mcp__emailable__verify_email({ email: "address@domain.com" })

Batch Verification:

mcp__emailable__verify_emails_batch({
  emails: ["email1@domain.com", "email2@domain.com", ...]
})

Response Interpretation

State Meaning Action
deliverable Email is valid and accepts mail ✅ Safe to send
undeliverable Mailbox does not exist ❌ DO NOT SEND - Remove from list
risky May bounce (catch-all, full mailbox) ⚠️ Send with caution
unknown Could not be verified ⚠️ Send with caution

Step 4: Find a Free Image (Pixabay)

Website: https://pixabay.com/

Pixabay provides royalty-free images that can be used commercially without attribution.

Search Tips

  1. Go to pixabay.com
  2. Search for relevant terms (e.g., "Christmas robot", "holiday technology")
  3. Filter by:
    • Type: Photos or Illustrations
    • Orientation: Horizontal (best for email headers)
    • Size: Large (for high quality)
  4. Download the image in the largest available resolution

Step 5: Host the Image (Cloudinary)

Why Cloudinary?

Email clients do not display local images. You must host images on a public CDN. Cloudinary provides free hosting with fast global delivery.

Step 6: Design the HTML Email

Template Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Email Subject</title>
</head>
<body style="font-family: Arial, sans-serif; line-height: 1.7;
             color: #121212; background-color: #f5f8fa;
             margin: 0; padding: 20px 0;">

    <!-- Container -->
    <div style="max-width: 600px; margin: 0 auto;
                background-color: #ffffff; border-radius: 12px;
                overflow: hidden;">

        <!-- Header Image -->
        <img src="YOUR_CLOUDINARY_URL" alt="Description"
             style="width: 100%; height: auto; display: block;">

        <!-- Content -->
        <div style="padding: 45px 40px; text-align: center;">
            <h1 style="color: #1d3642;">Headline Here</h1>
            <p>Your message content here.</p>
        </div>

        <!-- Footer -->
        <div style="background-color: #44919c; color: #ffffff;
                    padding: 25px; text-align: center;">
            <p>&copy; 2025 The Cozm</p>
            <a href="mailto:support@thecozm.com?subject=Unsubscribe">
                Unsubscribe
            </a>
        </div>
    </div>
</body>
</html>

Design Best Practices

  • Use inline CSS - Email clients strip <style> tags
  • Max width 600px - Optimal for most email clients
  • Use web-safe fonts - Fallback to system fonts
  • Include alt text - For images that don't load
  • Include unsubscribe link - Legal requirement

Step 7: Send Emails with Claude Code

No Python required! Claude Code can send emails directly using the SendGrid API. Simply describe what you want to send and Claude will handle the rest.

SendGrid Configuration

Setting Value
Authenticated Domainthecozm.com
Default Frombenjamin@thecozm.com

How to Send with Claude Code

Simply ask Claude Code to send your campaign. Example prompts:

"Send a test email to benjamin@thecozm.com using the HTML template
I just created. Use SendGrid API."
"Send the Christmas campaign email to all verified contacts
in my Google Sheet. Use 1 second delay between sends."
"Send the attached HTML email to these 50 recipients:
[paste list from Google Sheets]"

What Claude Code Does

  1. Reads your HTML email template
  2. Connects to SendGrid API using the configured API key
  3. Sends emails one-by-one with rate limiting
  4. Reports success/failure for each recipient
  5. Can update your Google Sheet with sent dates

Rate Limiting

Claude Code automatically adds a 1-second delay between emails to respect SendGrid limits and maintain good sender reputation.

Step 8: Test and Send

Testing Workflow

  1. Send test first: Ask Claude Code to send a test email to yourself
  2. Check your inbox - Verify the email renders correctly
  3. Check on mobile - Forward to your phone and verify
  4. Check spam folders - Ensure it's not being caught by filters

Full Send

Once testing is complete, ask Claude Code to send to your full recipient list:

"Send the email to all 'deliverable' contacts in my Google Sheet.
Skip anyone marked as 'undeliverable'. Report progress as you go."

Claude Code will report success/failure for each recipient in real-time.

Troubleshooting

Email Not Arriving

  1. Check spam/junk folder
  2. Verify SendGrid API key is correct in CLAUDE.md
  3. Check SendGrid dashboard for delivery status
  4. Ensure thecozm.com domain authentication is active

Image Not Displaying

  1. Verify Cloudinary URL is accessible in browser
  2. Ensure https:// (not http://)
  3. Check image alt text is meaningful

Claude Code Issues

  1. Ensure SendGrid API key is configured in your global CLAUDE.md
  2. If rate limited, wait a few minutes before retrying
  3. For large campaigns (500+), consider splitting across multiple sessions
